# Contributing to Larjango

Thank you for your interest in contributing to Larjango.

Larjango is an open-source community project focused on a three-process workflow for planning, building, reviewing, and deploying full-stack web projects.

## Ways to Contribute

You can help by:

- improving documentation
- suggesting ideas
- reporting bugs
- creating examples
- reviewing pull requests
- writing tests
- improving project structure
- helping other community members

## Three-Process Workflow

Larjango follows a simple workflow:

1. **Plan** — define requirements, architecture, roadmap, and tasks.
2. **Build** — implement features, examples, documentation, and tooling.
3. **Review & Deploy** — test, review, document, release, and maintain.

## Getting Started

1. Fork the repository.
2. Create a new branch:

```bash
git checkout -b feature/your-feature-name
```

3. Make your changes.
4. Commit your work:

```bash
git commit -m "Add your clear commit message"
```

5. Push your branch:

```bash
git push origin feature/your-feature-name
```

6. Open a pull request.

## Pull Request Guidelines

Before opening a pull request, please make sure:

- your change has a clear purpose
- your code or documentation is readable
- your pull request explains what changed and why
- related issues are linked when possible
- you are respectful in discussions and reviews

## Good First Contributions

Good first contributions include:

- fixing typos
- improving README sections
- creating examples
- adding documentation
- suggesting project structure
- creating beginner-friendly issues

## Community Behavior

All contributors are expected to follow the Code of Conduct.

Thank you for helping build Larjango.
