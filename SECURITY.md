# Security Policy

Thank you for helping keep Larjango secure.

## Supported Versions

Larjango is currently in early development. Security updates will apply to the latest version on the `main` branch unless a versioning policy is added later.

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public issue.

Instead, report it privately to the project maintainer through an appropriate private contact method or GitHub security reporting if enabled.

When reporting a vulnerability, please include:

- a clear description of the issue
- steps to reproduce it
- affected files or features
- possible impact
- any suggested fix, if available

## Response Process

The maintainer will review security reports and work on a fix as soon as possible.

Possible actions may include:

- confirming the vulnerability
- requesting more information
- preparing a patch
- publishing a security update
- documenting mitigation steps

## Responsible Disclosure

Please give the maintainers reasonable time to review and fix the issue before sharing details publicly.

## Scope

This policy applies to Larjango source code, documentation, examples, workflows, and related project resources.
